What is Zoho Catalyst?

Zoho Catalyst is a full-stack serverless development platform — it provides compute (serverless functions), data (Datastore), auth, file storage, and web hosting as managed services inside the Zoho ecosystem.

⚙️

Serverless Functions

Node.js / Java functions deployed as HTTP endpoints — no servers to provision. Auto-scales from zero, billed per invocation.

🗄️

Catalyst Datastore

Managed NoSQL/relational store for app data — replaces a separate database for both projects. Queried via Catalyst ZCQL from serverless functions.

🔐

Catalyst Auth

Built-in user authentication — Zoho (internal staff) or External Auth (portal clients). Eliminates per-seat licensing for external portal users.

🌍

Web Hosting + AppSail

Static frontend hosting + containerised backend runtime for the Client Portal. Custom domain support for branded experience.

Project 1 · Leave Management App

A standalone employee self-service leave portal built entirely on Zoho Catalyst — independent of Zoho People's admin interface, accessible from any device without a Zoho login.

📋

Apply & Track Leave

Employees submit leave requests, view approval status in real-time, and cancel pending requests — all without touching Zoho People's admin UI.

⚖️

Balance Visibility

Real-time leave balance dashboard shows accrued, used, and remaining days per leave type — synced from Zoho People via Catalyst serverless functions.

✅

Multi-level Approvals

Manager approval chain handled entirely within Catalyst functions — email notifications sent on submission, approval, and rejection via Catalyst Mail service.

📆

Holiday Calendar

Company holiday calendar loaded from Zoho People and displayed alongside the leave form — employees see public holidays before selecting dates.

Zoho Catalyst · Node.js

Technical Architecture — Leave App

Fully serverless — no backend servers, no database to manage. Catalyst handles scaling, auth, and persistence automatically.

Catalyst Serverless Functions Catalyst Datastore Catalyst Auth (Zoho login) Catalyst Mail Zoho People API sync Node.js runtime

Key design choice: Catalyst handles all leave records in its own Datastore and syncs approved/rejected status back to Zoho People via API — Catalyst is the write layer, People remains the HR system of record.

Project 2 · Client Portal

A branded external-facing web portal for Translation Excellence clients — project submission, real-time delivery tracking, file download, and invoice access — all without requiring a Zoho account.

📤

Project Submission

Clients submit translation requests (source files, language pair, deadline, notes) via a branded form. Submission auto-creates a CRM deal and a Zoho Projects entry.

🔭

Delivery Status Tracker

5-stage pipeline displayed in real-time: Received → In Translation → QA → Delivered → Invoiced. Status sourced live from Zoho Projects milestones.

⬇️

Secure File Downloads

Translated documents stored in Catalyst File Store — clients download their deliverables through time-limited signed URLs, never directly from Workdrive.

🧾

Invoice & Payment History

Invoice PDFs pulled from Zoho Books and surfaced in the portal — clients see outstanding and paid invoices without needing access to Zoho Books.

Zoho Catalyst · AppSail · External Auth

Technical Architecture — Client Portal

External clients authenticate via Catalyst External Auth — fully independent accounts, no Zoho subscription required. AppSail runs the containerised backend.

Catalyst AppSail Catalyst External Auth Catalyst File Store Catalyst Datastore Zoho CRM API Zoho Projects API Zoho Books API

Key design choice: Catalyst acts as an API aggregation layer — it reads from CRM, Projects, and Books and presents a unified view to clients, so no system needs to be exposed directly.

Why Catalyst over Zoho Apps?

Design decisions behind choosing Catalyst for these two use cases instead of extending Zoho People or building in Creator.

Consideration Native Zoho Apps Zoho Catalyst
External client accessRequires Zoho seats (cost per user)Catalyst External Auth — unlimited, free
UI customisationConstrained to app templatesFull custom HTML/CSS/JS frontend
BrandingZoho-branded interfacesCustom domain, client's brand identity
API aggregationNot possible across appsSingle Catalyst endpoint can call CRM + Projects + Books
Serverless costPer-user licensingPer-invocation billing — near-zero at SMB scale
Deployment speedApp-level config onlyCatalyst CLI — deploy in <10 seconds
